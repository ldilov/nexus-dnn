## Gotchas for next session (READ FIRST)

1. **Stale-read trap** — still present. After any background-agent run writes a file, the subsequent Edit MUST be preceded by a fresh Read. Grep after a batch of Edits to verify each landed.
2. **Web bundle is built + committed to `web/dist/`** — changes to source now require a `pnpm build` + commit, otherwise the host-side activation contract fails. The `.gitignore` has been updated to exempt `extensions/builtin/*/web/dist/**` but still ignore `*.map` and `.vite/`.
3. **Extension Rust workspace is isolated** — `rust/Cargo.toml` declares `[workspace]` (empty). Never remove. The host uses `edition2024` which needs Rust 1.85+; extension pins 1.84.
4. **Boundary audits are two scripts**, both must pass: `scripts/audit-runtime-boundary.ps1` and `extensions/builtin/emotion-tts/scripts/audit-boundary.ps1`. Run both after ANY change.
5. **No code comments** (constitution Principle IV). The handful of module-level docstrings and the user-visible `OVERRIDE_DOCS` UI string are allowed. Everything else self-documents.
6. **Extensions stay decoupled from host** (feedback memory `feedback_extension_decoupling.md`). Builtin is a distribution channel, not an architectural license. Before importing any host code, think "could an external extension do this the same way?" — answer must be yes.
7. **`torch.compile` Windows-CUDA13** — spec 034 US4 is opt-in with fallback specifically because the Triton wheel is flaky on this platform. Don't force-enable in tests.
8. **RNNoise wheel** — spec 034 US1 assumes it *might* fail to install. The pipeline has a skip-denoise fallback by design. Don't add a hard dependency.
9. **Manifest additive-only** — every new field is `Option<T>` with `skip_serializing_if`. Old readers must continue to parse old exports.
10. **Spec-034 `/speckit.tasks` next** — already scaffolded, ordered plan in `quickstart.md`. Running `/speckit.tasks` will emit tasks.md aligned to steps 1-8.
